# CHANGELOG

すべての変更履歴をここに記録します。

## [1.5.1] - 2026-06-30

### 修正
- **Rust 1.96.0 および Edition 2024 に伴う Clippy 警告の解消**:
  - GUI アプリ (`src/bin/mynkf-gui.rs`) において、`add_file_paths` およびファイル追加ボタンの処理における二重の `if`・`if let` ネストを、Edition 2024 で安定化された `&& let` 構文 (let_chains) に結合してコードを整理しました。
  - GUI アプリの改行コード判定部分の冗長な `match` 式を `matches!` マクロに置き換えて簡潔な 1 行にリファクタリングしました。
  - GUI アプリの Windows API 独自の命名規則に起因する `upper_case_acronyms` 警告を、`#[allow(clippy::upper_case_acronyms)]` アトリビュートを `win32` モジュールに付与することで抑制しました。
  - ライブラリ (`src/lib.rs`) において、22 箇所の Clippy 警告（`manual_range_contains` による `(start..=end).contains(&x)` への推奨書き換え、`collapsible_if` の結合、`manual_flatten` の適用など）を全面的に修正・解消しました。
- **ドキュメント改善とステータスバッジの追加**:
  - `README.md` および `README.ja.md` のタイトル直下に、GitHub Actions の CI/Release ビルドステータスバッジを追加しました。
  - `.agents/AGENTS.md` の CI/CD 維持ルールに、ステータスバッジの表示を維持する旨のガイドラインを追記しました。
- **バージョン表示の同期化**:
  - `Cargo.toml` のバージョン設定を `1.5.1` に更新しました。
  - `src/main.rs` (CLI版) のヘルプ表示内で古いバージョン番号（`v1.4.0`）がハードコードされていた箇所を、`env!("CARGO_PKG_VERSION")` を使用して `Cargo.toml` と自動同期するように修正しました。

## [1.5.0] - 2026-06-29

### 追加
- **GUI版文字コードコンバータの新規追加 (`mynkf-gui`)**:
  - `eframe`/`egui` を使用した、超軽量・高速動作のデスクトップGUIアプリを追加しました。
  - **二重起動防止**: Windows上で多重起動した際、後から起動したプロセスが即座に正常終了するように実装。
  - **枠なし・影なしの完全フラットデザイン**: Windowsの DWM (Desktop Window Manager) API を直接制御し、ウィンドウの枠と影を完全に排除。
  - **カスタムタイトルバー**: 自作ヘッダーを設け、ドラッグ操作によるウィンドウ移動に対応。
  - **ファイル一括変換**: ファイルのドラッグ＆ドロップまたはファイルダイアログ選択により、現在の文字コード/改行コードを表示し、一括で変換保存が可能。
  - **テキスト直接変換**: テキストの貼り付け、リアルタイムでの変換結果（文字コード/改行コード）のプレビュー、およびクリップボードコピーに対応。
- **コアロジックのライブラリ化 (`src/lib.rs`)**:
  - CLI版とGUI版のコード共有のため、文字コード判定や変換処理などの主要ロジックを `mynkf` ライブラリ（`src/lib.rs`）として抽出しました。
- **Windows-sys 依存の排除とダイレクト FFI**:
  - GUI版のバイナリサイズとビルド時間を最小限に抑えるため、`windows-sys` への依存を排除し、二重起動防止やDWM制御に必要な Win32 API 宣言を自前で定義する直接 FFI 構成を適用しました。

### 修正
- **改行コードのみの変換サポート (元の文字コードの維持)**:
  - これまでは文字コード変換オプションが未指定の場合にデフォルトで UTF-8 へ変換されていましたが、文字コード指定オプション（`-w`, `-s`, `-e`）を何も指定せず、`-d` (LF) または `-c` (CRLF) のみを指定した場合に、入力ファイルの文字コード（自動判定結果）を維持したまま改行コードのみを変換するよう挙動を修正しました（本家 `nkf` の挙動に準拠）。
- **GUI版カスタムタイトルバーのドラッグ干渉修正と操作範囲拡張**:
  - ウィンドウ全体のドラッグ移動用の判定エリアが閉じる `[X]` および最小化 `[-]` ボタンの上に覆いかぶさり、クリック反応を妨害していたバグを修正しました。
  - ドラッグ用の判定領域を背面レイヤーへ配置し、その前面にボタンやテキストを描画する構造（`allocate_ui_at_rect`）へ変更しました。これにより、ボタンクリックの100%の応答性を確保しつつ、タイトルラベルのみならずタイトルバーの「余白部分」全体を掴んでウィンドウをドラッグ移動できるよう操作性を劇的に向上させました。
  - GUI版のタイトルバーへ、アプリ名とともにバージョン番号（例: `v1.5.0`）を表示するラベルを追加しました。
- **バージョン情報の動的取得とハードコーディング排除**:
  - CLI版 (`mynkf`) でハードコーディングされていた `"MyNKF v1.4.0"` などの文字列を廃止し、Rustのマクロ `env!("CARGO_PKG_VERSION")` を使って `Cargo.toml` の `version` フィールドから動的にバージョン情報を取得して表示する形に変更しました。これにより、GUI版と合わせてバージョン情報の一元管理が可能になりました。
- **未使用のテーブル引数の完全削除**:
  - ライブラリ内 (`src/lib.rs`) の `encode_from_unicode` 関数において、以前の変換ロジックで使われていたものの現在は不要になっていた `table` 引数を関数定義、およびすべての呼び出し元（CLI版、GUI版、テストモジュール）から完全に削除し、コードのクリーン化を行いました。

## [1.4.0] - 2026-06-29

### 追加
- **`--size` オプション（ファイルサイズ表示）**:
  - 自動判定（`--guess`）モード時、ファイルサイズも併せて表示する機能を追加しました（例: `UTF-8 (LF) [1.2 KB]`）。
  - ファイルサイズはデータ量に応じて適切に単位（B, KB, MB, GB）を自動選択してフォーマット表示します。
  - テキストファイルだけでなく、文字コード判定結果が `BINARY`（Unknown）の場合もサイズを表示します（例: `BINARY [1.2 KB]`）。
  - 行数カウントオプション（`--line`）との併用も可能です（例: `UTF-8 (LF) [123 lines] [1.2 KB]`）。

## [1.3.0] - 2026-06-29

### 追加
- **コマンドライン引数のワイルドカード（グロブ）自動展開**:
  - Windows環境（PowerShellやcmd.exe等）など、シェル側で引数のワイルドカード展開が自動で行われない環境でも、プログラム自身が `*` や `?` を適切に展開して複数ファイルを処理する機能を実装しました（外部依存なし）。
  - 大文字・小文字を区別せずにパターンマッチングを行います。
- **処理ファイル数上限の制限**:
  - 低リソース環境下での過度なCPU・メモリ負荷を避けるため、一度に処理可能なファイル数を最大 **100ファイル**（グロブ展開後を含む）に制限しました。上限を超えた場合は、エラーメッセージを出力し、ステータスコード `1` で即座に終了します。

---

## [1.2.0] - 2026-06-29

### 追加
- **改行コードの自動判定表示**:
  - `--guess`（`-g`）オプション実行時、非 `BINARY` ファイルであれば改行コードの種類（`LF` / `CRLF` / `CR` / `MIXED` / `NONE`）も併せて表示する機能を追加しました（例: `UTF-8 (LF)`）。
- **`--line` オプション（行数カウント）**:
  - `--guess` モードと併用する `--line` オプションを追加しました。指定された場合、テキストファイルの論理行数も併記します（例: `UTF-8 (LF) [120 lines]`）。
  - 文字コード判定が `BINARY`（Unknown）の場合は、改行コードや行数の情報は出力されず、単に `BINARY` と出力されます。

## [1.1.1] - 2026-06-29

### 追加
- **自動単体テスト**:
  - `src/main.rs` に 11 個の自動単体テスト（`cargo test`）を実装しました。
  - 文字コード自動判定（guess）、座標変換、文字コード変換、フォールバック、半角カタカナの挙動をカバー。
- **エディタ設定ファイル**:
  - コードフォーマットやエンコーディング（BOMなしUTF-8、改行LF）を統一するため、`.editorconfig` および `.vscode/settings.json` を新規作成しました。
- **CI/CD 自動化設定**:
  - プッシュ・PR時に自動テストとビルド検証を行う `.github/workflows/ci.yml` を追加。
  - タグプッシュ時に Windows 向け CLI バイナリをパッケージングし自動リリースする `.github/workflows/release.yml` を追加.
  - 依存ライブラリの週次アップデートを追従する `.github/dependabot.yml` を追加。

### 修正
- **EUC-JP -> Shift_JIS 座標変換バグの修正**:
  - `eucjp_to_sjis` 関数において、第1バイトと第2バイト of 座標変換定数（それぞれ `0x70` を `0x80` へ修正、`ten + 0x7D` などを正しいJIS座標の対応式へ修正）の誤りを正し、正確な Shift_JIS 出力が行われるように修正しました。
- **名称の統一**:
  - 生成プログラムが `MyNKF.exe` になるため、ソースコード（ヘルプ/バージョン出力等）や各種ドキュメント、リリース設定内に点在していた `nkf-win` / `NKF-Win` の表記を `MyNKF` へ完全に変更・統一しました。
- **未知のオプションに対するエラー終了処理の追加**:
  - サポートされていないオプション（`-` から始まる未知の引数）が指定された際、以前は無視されて標準入力のパイプ待ち（無応答状態）になっていましたが、エラーと使用方法を出力した上で、即座に終了ステータス `1` で正常にエラー終了するように修正しました。
- **仕様書およびドキュメントの補足**:
  - 本家 `nkf` との完全な互換性を維持するために定義されている `--versio` オプションの背景（独自引数パースによる前方一致の手動エミュレーションまたはタイポ救済）について、`README.ja.md`、`README.md`、および `docs/SPEC.md` に注釈を追記しました。

## [1.0.0] - 2026-06-28

### 追加
- **MyNKF Rust 互換ユーティリティの初期リリース**:
  - 100% Rust 標準ライブラリのみで実装された `nkf` 互換ツール。
  - UTF-8、Shift_JIS、EUC-JP の相互変換、自動判定（--guess）をサポート。
- **Webデスクトップシミュレータの実装**:
  - Windows環境を模したGUIシミュレータ（常に手前に表示、タイトルバー非表示、背景透過をエミュレート）。
  - ドラッグ＆ドロップによるファイルアップロード、変換結果の即時ダウンロード、CLIでのコマンド実行機能。
- **Obsidian連携用ドキュメントエクスポート**:
  - Rustの全ソースコード、ライセンス、仕様書などをObsidianにそのまま貼り付け可能な形式で出力・コピー・エクスポートする機能。
