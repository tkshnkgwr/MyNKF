# 仕様書 (SPEC.md)

本ドキュメントは、低リソース環境向けに最適化された `MyNKF` およびその Web シミュレータの最新の機能仕様を定義するものです。

---

## 1. 概要
`MyNKF` は、伝統的な日本語文字コード変換ツール `nkf` (Network Kanji Filter) の機能を、100% Rust 標準ライブラリ（`std`）のみでエミュレートした軽量なコマンドラインユーティリティです。外部依存ライブラリ（Crates）を一切持たず、セキュリティ、ポータビリティ、およびリソース消費（CPU、メモリ、バイナリサイズ）に極限まで配慮して設計されています。

また、Web上で動作を擬似体験できる「Webデスクトップシミュレータ」も同時に実装されており、Windows のタスクバーやウィンドウ（常時最前面、背景透過）などを模した環境で CLI やファイルの文字コード変換をテストできます。

---

## 2. 動作環境・制約
- **ターゲットOS**: Windows 10 / 11（低リソースPCを想定、Rustビルドはクロスプラットフォーム動作可能）
- **依存関係**: 
  - **CLI版 (`mynkf`)**: 外部クレート依存なし (純粋な Rust `std` のみ使用)
  - **GUI版 (`mynkf-gui`)**: GUIライブラリ `eframe`/`egui` およびファイルダイアログ `rfd` を使用。Windows Native API に関しては `windows-sys` などの巨大クレートを導入せず、自前で `extern "system"` 宣言（直接 FFI）を行うことで、バイナリサイズとコンパイル時間を最小化。
- **バイナリサイズ目標**: 
  - CLI版：`cargo build --release` 時に `strip` オプション等を適用し、約 200KB〜250KB。
  - GUI版：サイズ優先の最適化およびストリップを適用し、グラフィック機能内蔵でありながら数MB（3〜5MB程度）の極小サイズに抑える。
- **メモリ使用量**:
  - CLI版：数MB以下（ストリームまたはバッファ処理）。
  - GUI版：数MB〜十数MB（eguiによる描画バッファ）。低メモリPCでも問題なく常駐動作が可能。
- **処理ファイル数制限**:
  - メモリやCPUへの過度な負荷を防止するため、一度に処理可能なファイル数は最大 **100ファイル**（ワイルドカード展開後を含む）に制限されます。上限を超えた場合はエラーを出力して終了します。

## 3. CLI コマンド仕様

### 3.1 起動引数と構文
```powershell
MyNKF [options] [file...]
```

### 3.2 サポートするオプション
| オプション | エイリアス | 説明 |
| :--- | :--- | :--- |
| `-w` | `--utf8` | 出力を UTF-8 に変換する。改行コードは原則 `LF`。 |
| `-s` | `--sjis` | 出力を Shift_JIS に変換する。改行コードは原則 `CRLF`。 |
| `-e` | `--euc` | 出力を EUC-JP に変換する。改行コードは原則 `LF`。 |
| `-g` | `--guess` | 指定されたファイル、または標準入力の文字コードを判定し結果を表示する。 |
| | `--line` | 自動判定（`--guess`）モード時、テキストの論理行数も併せて表示する（`BINARY` を除く）。 |
| | `--size` | 自動判定（`--guess`）モード時、フォーマットされたファイルサイズも併せて表示する（例: `[1.2 KB]`, `[100 B]`、`BINARY` の場合も表示）。 |
| `-d` | `--lf` | 改行コードを強制的に `LF` に変換する。 |
| `-c` | `--crlf` | 改行コードを強制的に `CRLF` に変換する。 |
| `-h` | `--help` | ヘルプテキストを表示し、終了する。 |
| `-v` | `--version`, `--versio` | バージョン情報 (`v1.5.0`) を表示し、終了する。 |

> [!NOTE]
> `--versio` （最後の `n` が欠けた形式）は、本家 `nkf` のコマンドライン引数解析の挙動（独自パーサーによる前方一致のエミュレーションや歴史的なタイポの救済措置）と完全に一致させるために定義されている互換オプションです。

- **文字コード指定オプションが無い場合の挙動 (改行コードのみの変換)**:
  - 文字コード変換オプション（`-w`, `-s`, `-e`）のいずれも指定しなかった場合は、入力データの文字コード（自動判定結果）がそのまま出力ファイルの文字コードとして維持されます。
  - これにより、`-d` (LF) または `-c` (CRLF) オプションのみを指定した際、入力のエンコーディングを変更することなく、改行コードのみを変更して出力することができます（本家 `nkf` 準拠）。
- **ファイル引数の省略**: ファイルパスが省略された場合は、標準入力 (`stdin`) を対象とし、処理結果を標準出力 (`stdout`) に送ります。
- **複数ファイル指定**: 複数のファイルを引数に渡した場合、各ファイルが順番に処理されて標準出力へ出力されます。
  - **ワイルドカード対応**: 引数にはワイルドカード（`*`、`?`）を使用できます（例: `MyNKF --guess *.txt`）。Windowsの標準コマンドプロンプトやPowerShell環境など、シェルによる展開が行われない環境においても、プログラム自身が簡易的なグロブ展開を行います。
  - **最大件数制限**: 安全のため、展開後の合計ファイル数が **100ファイル** を超える場合は、エラーを表示して処理を即座に中断します。

## 4. 文字コード変換および判定ロジック

### 4.1 自動判定（Guess）
入力バイト列を走査し、エンコーディングの特徴量（各エンコードに特徴的なマルチバイトシーケンスの整合性）に基づいて以下の優先順位で文字コードを推定します。
1. **ASCII**: すべてのバイトが `0x00`〜`0x7F` の範囲内にある場合。
2. **UTF-8**: 有効な UTF-8 マルチバイトシーケンス（2〜4バイト長、`0x80`〜`0xBF` の継続バイト等）のみで構成されている場合。
3. **EUC-JP**: EUC-JP の2/3バイトの有効範囲（例: `0xA1..=0xFE`）に適合する場合。
4. **Shift_JIS**: Shift_JIS の第1バイト（`0x81..=0x9F`, `0xE0..=0xFC`）および第2バイト（`0x40..=0x7E`, `0x80..=0xFC`）、または半角カタカナ（`0xA1..=0xDF`）のパターンに適合する場合。
5. **BINARY**: 明らかな制御文字や非テキスト文字が含まれている、または上記いずれのテキストエンコーディングのルールにも適合しない場合。

#### 自動判定出力フォーマット
自動判定（`--guess`）時の出力は、入力データおよびオプションの有無により以下のように変化します。
- **`BINARY` の場合**:
  改行コードおよび行数の追加表示は行いません（ファイルサイズは表示可能です）。
  - 標準入力: `BINARY`
  - 標準入力（`--size` あり）: `BINARY [1.2 KB]`
  - ファイル指定時: `file.bin: BINARY`
  - ファイル指定時（`--size` あり）: `file.bin: BINARY [1.2 KB]`
- **`BINARY` 以外（テキスト）の場合**:
  判定された改行コード（`LF` / `CRLF` / `CR` / `MIXED` / `NONE`）を併記します。`--line` 指定時は論理行数、`--size` 指定時はファイルサイズがそれぞれ併記されます。
  - 標準入力: `UTF-8 (LF)`
  - 標準入力（`--line` あり）: `UTF-8 (LF) [123 lines]`
  - 標準入力（`--size` あり）: `UTF-8 (LF) [1.2 KB]`
  - 標準入力（`--line` と `--size` あり）: `UTF-8 (LF) [123 lines] [1.2 KB]`
  - ファイル指定時: `file.txt: UTF-8 (LF)`
  - ファイル指定時（`--line` あり）: `file.txt: UTF-8 (LF) [123 lines]`
  - ファイル指定時（`--size` あり）: `file.txt: UTF-8 (LF) [1.2 KB]`
  - ファイル指定時（`--line` と `--size` あり）: `file.txt: UTF-8 (LF) [123 lines] [1.2 KB]`

### 4.2 マッピングと外字フォールバック
- **JIS X 0208 基準のマッピング表**: Rustの標準コードとして配列形式でエンコーディングマッピングを埋め込んでいます。
- **半角カタカナの保持**: Shift_JIS の `0xA1`〜`0xDF`（半角カタカナ）は、UTF-8 や EUC-JP の対応するコードへ正しくマッピング変換され、喪失しません。
- **フォールバック**: マッピングが定義されていない外字や特殊文字が検出された場合、エラーにせず `??`（2つのクエスチョンマーク）に置換して処理を継続します。

## 5. デスクトップシミュレータ仕様
Webブラウザ上で動作するシミュレータは、以下の性質を備えています。
- **ビジュアルデザイン**: Impact風の力強いタイポグラフィ、縦長コンパクト表示、常時最前面ウィンドウ。背景透過エミュレーションおよび枠なしウィンドウ。
- **入力・ドラッグ＆ドロップ**: テキスト入力だけでなく、ファイルを直接ドラッグ＆ドロップして、文字コードの判定・変換を行えます。
- **CLIシミュレーション**: 画面下部の擬似シェルから `MyNKF` コマンドを実行し、結果を画面に出力できます。
- **ダウンロード連携**: 変換したファイルを指定した文字コードとファイル名で即座にローカルダウンロード可能です。
- **Obsidianエクスポート**: シミュレータから最新ソースコードやドキュメントを Markdown 形式で一括でコピー・ダウンロードするインターフェースを提供します。

## 6. デスクトップGUIアプリケーション仕様 (`mynkf-gui`)
デスクトップGUI版は、`eframe`/`egui` フレームワークを用いた常駐可能・高速動作のユーティリティです。以下の仕様および制御を特徴とします。

### 6.1 二重起動の防止
- **仕様**: Windowsシステムにおいて本GUIアプリケーションが多重起動されるのを防止します。
- **制御ロジック**: `CreateMutexW` によるシステム広域 Named Mutex `Global\MyNKF_GUI_SingleInstance_Mutex` の所有を試み、既に Mutex が存在して `GetLastError() == ERROR_ALREADY_EXISTS` と判定された場合は、新しく起動されたプロセスが直ちに正常終了（exit code 0）します。

### 6.2 ウィンドウ表示と枠・影の排除
- **仕様**: Windowsのネイティブフレーム（タイトルバー、リサイズ枠、および境界線の影）を完全に非表示にし、独自の透過フラットデザインを実現します。
- **制御ロジック**:
  - `eframe::NativeOptions` の `viewport` を用いて、`decorations: false`（装飾なし）および `transparent: true`（背景透過）を指定します。
  - アプリケーション初期化の最初のフレームにおいて、Win32の DWM (Desktop Window Manager) API である `DwmSetWindowAttribute` を呼び出し、`DWMWA_NCRENDERING_POLICY` を `DWMNCRP_DISABLED` に設定して、Windowsシステムが提供するデフォルトのウィンドウ影や薄い枠を完全に無効化します。

### 6.3 カスタムヘッダーとウィンドウ移動
- **仕様**: ネイティブのタイトルバーが無い代わりに、UIの最上部に自作のヘッダーバーを設け、閉じる `[X]`、最小化 `[-]` 操作およびウィンドウのドラッグ移動を提供します。
- **制御ロジック**:
  - ヘッダーバー領域のドラッグ入力を検知（`response.dragged()`）した際、`ctx.send_viewport_cmd(egui::ViewportCommand::StartDrag)` を呼び出し、OS標準のネイティブウィンドウドラッグ移動処理へ処理を委譲します。これにより、ラグや描画の崩れのない滑らかな移動を実現します。
  - 閉じるボタンがクリックされた際は `ViewportCommand::Close`、最小化の際は `ViewportCommand::Minimized(true)` を送信します。

### 6.4 日本語表示とフォントの動的ロード
- **仕様**: さまざまなWindows環境で文字化け（豆腐化）せず日本語テキストが表示できるよう、システムフォントを動的に参照します。
- **制御ロジック**:
  - コンパイル時に特定のフォントファイルを埋め込む（`include_bytes!`）のではなく、実行時に `C:\Windows\Fonts\` ディレクトリから `meiryo.ttc` / `msgothic.ttc` / `msmincho.ttc` をこの優先順位で走査して読み込み、`egui::FontDefinitions` に動的に登録します。これにより、フォント非搭載環境でのビルド崩壊を防ぎながら、実行環境における日本語表示の堅牢性を確保します。

### 6.5 ファイル一括変換機能
- **ドラッグ＆ドロップ**: アプリケーションウィンドウの任意の領域にファイルをドラッグ＆ドロップすることで、変換リストに自動でファイルを追加します。
- **ファイルピッカー**: `rfd` ファイルダイアログを用いて、個別に複数ファイルを選択してリストに追加できます。
- **自動解析表示**: リストに追加された各ファイルに対して、ライブラリ化したコアロジックを走査し、「ファイルサイズ」「現在の文字コード（UTF-8 / Shift_JIS / EUC-JP / BINARY等）」「改行コード（LF / CRLF / CR 等）」を即座に自動判定して表形式で表示します。
- **一括上書き保存**: 出力文字コードおよび改行コード（LF / CRLF / 元のまま）を指定し、一括でファイルを変換して同名ファイルへ上書き保存します。変換処理後はステータス（「完了」等）や更新後のサイズ・エンコーディングをリアルタイムにUIに再反映します。

### 6.6 テキスト直接変換機能
- **プレビューと変換**: 入力されたテキストに対してリアルタイムに指定したエンコーディングによる変換シミュレーションを行い、結果をクリップボードにコピーできます。
- **文字コード再現性**: クリップボードにコピーされる際、指定した文字コード（UTF-8, Shift_JIS, EUC-JP）で一度エンコードされたバイト列を模擬デコードし、改行コードのフォーマットも含めて忠実に再現された文字列が転送されます。
