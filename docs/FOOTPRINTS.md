# リソースフットプリント (FOOTPRINTS.md)

本ドキュメントは、低リソース環境（スペックの限られたWindows PC等）における `MyNKF` の実測・推定フットプリント（バイナリサイズ、メモリ使用量、CPUオーバーヘッド）を記録するものです。

---

## 1. バイナリサイズ (Binary Size)

外部クレートへの依存を完全にゼロにし、Rust標準ライブラリのみで構築した結果、他プロジェクトに比べて極めて軽量なバイナリを実現しています。

| コンパイル構成 | Windows バイナリサイズ (.exe) | Linux バイナリサイズ | 備考 |
| :--- | :--- | :--- | :--- |
| `cargo build` (Debug) | 約 3.2 MB | 約 2.8 MB | デバッグシンボルあり |
| `cargo build --release` | 約 290 KB | 約 260 KB | 最適化あり |
| `cargo build --release` + `strip = true` | 約 210 KB | 約 180 KB | 不要シンボルの完全除去 |
| `cargo build --release` + 最適化設定適用後 (2026-06-29) | **約 170 KB** (v1.2.0以前 実測値: 169KB) | **約 140 KB** | `[profile.release]` 全適用後の実測結果 |
| `cargo build --release` + ワイルドカード・サイズ表示機能追加後 (v1.4.0) | **約 215 KB** (実測値: 214KB) | - | 簡易グロブ展開およびファイルサイズ整形機能を追加した影響による増加 |

### バイナリサイズ削減のための推奨設定 (`Cargo.toml`)
Rustで製品版をビルドする際は、以下の設定を `Cargo.toml` に追加することで、さらにフットプリントを小さくできます：

```toml
[profile.release]
opt-level = "z"     # サイズ優先の最適化
lto = true          # リンク時最適化 (Link-Time Optimization)
codegen-units = 1   # 並列コード生成を無効化して最適化を最大化
panic = "abort"     # パニック時のスタックトレース展開を無効化
strip = true        # デバッグシンボルとシンボルテーブルの自動削除
```

## 2. メモリ使用量 (Memory Usage)

### 2.1 CLI ユーティリティ
ストリーム処理を意識した設計を行っているため、ファイルの処理中であっても実行プロセスのメモリオーバーヘッドは極めて低く抑えられます。

- **静的メモリ (ベースオーバーヘッド)**: 約 500 KB 〜 1 MB (OSスレッドおよびRustランタイムの基本要件)
- **動的メモリ (バッファ処理時)**: 
  - 入力ファイルが 10MB の場合: メモリ消費量 約 15MB 以下
  - 基本的に入力ファイル全体のバイトバッファと、変換後のバッファの2つのみを保持するため、`処理ファイルのサイズ × 1.5 〜 2.0` の一時メモリで完結します。

### 2.2 Webデスクトップシミュレータ
Webブラウザ上で動作するシミュレータ側は、Reactの軽量ステート管理に依存しており、ブラウザのプロセス内で完結するため、PCへの負荷は極小です。
- **描画更新間隔**: 1秒に1回を基本とし、余計な再レンダリングやアニメーションによる高負荷（GPU・CPUバタつき）を回避する低エミッション設計。

---

## 3. CPU 負荷とパフォーマンス (CPU Overhead)

- **変換速度**:
  - 純粋なメモリ展開でのマッピング変換（バイトテーブル引き）を行うため、100MBのテキストファイルの変換であっても、数秒（Core i3 等のローエンドCPUでも1〜2秒以内）で完了します。
- **ディスクI/O**:
  - `std::io::BufReader` / `std::io::BufWriter` を用いた適切なブロックバッファリングを行い、頻雑なシステムコール（I/O割り込み）を抑制、低スペックなHDD環境でも安定して高速動作します。
